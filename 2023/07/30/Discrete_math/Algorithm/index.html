算法

这一章我们来讲讲算法的一些概念和例子，算法这一来源来自问题的一般性，简单来说：例如寻找序列 101,102,13,888中最大的数字。实际上就是寻找最大整数的一个特例，我们的问题就是如何解决这种问题，并解决这个问题中的一般性问题，为此我们必须给出一个算法，它指定了一系列的步骤来解决这个一般性问题。

为此不妨给出一个定义：

定义1

算法是进行一项计算或解决一个问题的准确指令的有限序列

类似于数学归纳法，但不同的是这里指的是指令，现在让我们解决开头说的问题。

冒泡排序

这是一个最简单的排序算法，但不是最有效的，冒泡算法通过连续比较相邻的元素，如果顺序不对就交换相邻的元素，从而把一个表的排列变成升序。

插入排序

插入排序比较简单，但是是从第二个元素开始的。插入排序将第二个元素与第一个元素作比较，如果不大于第一个元素，则插入到第一个元素的位置，如果大于第一个元素则把他插入到第二个元素后面。若顺序正确，则把第三个元素和第一个元素比较。若大于第一个元素，则继续与第二个元素比较。它将插入到第三个元素的正确位置上。

贪婪算法

我们接下来要说的算法几乎用于解决最优化问题（optimization problem）这种问题的目标一半是寻找给定问题的某个参数值最小或最大化的解。一个简单最优化问题就是小学接触过的围墙问题，给定一定数量的栅栏，求栅栏能围出的最大面积是什么。

令我们诧异的是，一种最简单的方法常常可以是最优化问题中的一个解，这种方法在每一步都选择最好的选项，而不受通盘考虑可以导致最优解的全部步骤序列，这种每一步都选择看起来“最好的”选项的算法我们叫贪婪算法。

停机问题

停机问题告诉我们，是否存在一个程序，能够判断另一个程序在特定的输入下是否会给出结果（停机）或者是无限的执行下去。

这个问题是不可解的，而证明是图灵在1936年给出的

我们假设有这么一个程序K(P)，其中P是过程P(H,K)。H是程序，K是输入。其中过程P只有在输入H的时候输出停机，否则无限循环。而程序K(P) 在P输出停机的时候输出无限循环，反之亦然。

现在，若P输出无限循环，则K(P)停机，这时候，因为K(P)停机。但我们把K作用到自己身上呢？想想K(K)，当K(P) = 停机，则K(K)无限循环，这时候K应该是输出停机，而K(K)应该是无限循环，矛盾。反过来也一样，假设K(K) = 无限循环，则这个时候应该输出K(K) = 停机，矛盾。

所以这个问题是无解的。

函数的增长

就像我们在微积分中接触的$o(x^2)$是一样的，我们要来谈谈函数的增长速度。

在上一节我们谈论了算法的概念，介绍了解决各种问题的算法，包括一些在列表中搜索元素和列表进行排序。注意的是，当我们估计线性搜索和二分搜索算法在n个元素的序列中搜索元素时用的比较次数。对于排序算法，冒泡排序和插入排序我们要算对n个元素的列表进行排列所需要的比较次数。这意味着我们解决此类问题的时候所需要的时间不仅仅取决于所用的操作步骤，这个时间还取决于实现这算法的程序硬件和软件。

在等一下我们要谈到的大O记号（big-O natation） 有一个非常好的作用，就是可以估计一个函数的增长而不用担心常数因子或低阶项，这意味着使用大O记号不用担心实现算法所用的硬件和软件。另外我们还可以利用大O记号假设算法的不同操作都花费了相等的时间，这大大的简化了分析。

大 O 记号

我们在微积分中使用了这么一个符号描述了函数的增长

定义1

令$f,g$是两个函数，其中$f:R\to R, g:R\to R$。如果存在常数C和k使得主要当$x>k$有

$\mid f(x)\mid \leq C\mid g(x)\mid$

我们就说$f(x)$是$O(g(x))$的，即$f(x)$最多不超过函数$g(x)$的。

为了利用大O记号，则我们需要选择k的值使得当$x>k$的时候容易估计$\mid f(x)\mid$的大小，并查看是否能找到一个常数C使得当$x>k$有$\mid f(x)\mid < C\mid g(x)\mid$

例1

我们来证明 $f(x) = x^2+2x+1 $是$O(x^2)$的

证明 容易看出$2x < x^2$和$11$就有

$f(x) \leq g(x)$

所以，只需要选择$C=4$和$k=1$就能够简单的证明$f(x)$是$O(x^2)$的。或者我们用更严格的方法。

记$f(x) = x^2+2x+1$和$g(x)= 3x^3$，我们计算

$\lim\limits_{x\to 1}\frac{x^2+2x+1}{3x^2} = 1$

这说明在$x=1$的时候$f,g$是同阶无穷小。

例2

证明$n^2$不是$O(n)$

如果要用定义证明，我们必须选择$C,k$不满足定义1，即证明不等式

$n^2> Cn$

那么构造$f(n) = n^2$和$g(n) =Cn, C\in R$。

利用函数$F(n) = n^2-Cn$，求导得到$F’(n) = 2n-C$，C是一固定常数，则在某个点$n\in (C,\infty)$上$F’(n) < 0$，因此$n^2$总能找到比$Cn$大的点。所以$n^2 > Cn$。

一些重要函数的大O估算

我们来看一些比较重要的函数，例如一般的多项式。

定理 1

令$f(x) = a_nx^n+a_{n-1}x^{n-1}+\cdots +a_1x+a_0$，其中$a_0,\cdots, a_{n} \in R$，那么$f(x)$是$O(x^n)$的。

证明 利用三角不等式，当$x>1$有

$\begin{aligned} \mid f(x)\mid =& \mid a_nx^n+\cdots+a_1x+a_0\mid \\ \leq& \mid a_n\mid x^n+\cdots+\mid a_1\mid x+\mid a_0\mid\\ =& x^n(\mid a_n\mid+\mid a_{n-1}\mid/x+\cdots + \mid a_1\mid /x^{n-1}+\mid a_0\mid/x^n)\\ \leq &x^n(\mid a_0\mid +\cdots + \mid a_1\mid+\mid a_0\mid) \end{aligned}$

证毕。

n!的大O估计

我们定义阶乘函数$f(n)=n!$，其中$n$是正整数。为了得到一个n!的大O估计，有

$\begin{aligned} n! =& 1\cdot 2\cdots n\\ \leq& n\cdot n\cdots n\\ =& n^n \end{aligned}$

所以，$n!$是$O(n^n)$的

$\ln( n)$的大O估计

利用$n< 2^n$，则容易得到$n$是$O(2^n)$的。那么我们可以这样子干，我们在不等式两边风别取$2$为底的对数

$\begin{aligned} &\log_2(n) < n\log_2(2)\\ =& \log_2(n) < n \end{aligned}$

更一般的，为了证明$\ln(n)$，有

$\ln(n) < n\ln(2)$

所以$\ln(n)$是$O(n)$的。
