数学基础

估算算法复杂度是一个理论问题，所以我们需要一套系统构架，先从下述的几个数学定义开始：

定义1： 令$c,n_0$是一些正常数，再令$f,T$是两个函数，这时若存在一个常数$N\geq n_0$使得$T(N) \leq cf(N)$，则我们说$T(N) = O(f(N))$

现在，给定函数$f(x) = x^2, T(x) = x$是两个函数。我们断言一定有$T(x) =O(f(x)) $，但怎么证明呢，定义让我们寻找出一个常数$c$和常数$n$，只要在数轴的某个数$N$之后有$T \leq cf$就可以了。现在，对$x^2$，我们知道在$0< x< 1$的时候有$x > x^2$，所以我们不应该在这个区间内去验证定义，我们选大点的数字，例如$N = 10000$,那么对$N$之后的数字$x$一定有$x < x^2$。所以这个时候我们可以说$T(3x) = O(f(x))$。

注意：该定义反过来是不一定成立的，注意到对$x$，只需要构造函数$3x$。我们只需要找任意大于3的常数$c$，就能得到$3x < cx$，就有$3x = O(x)$。所以该定义不是一个严格的偏序定义。等会我们将继续分析和给出更严格的定义。

定义2： 若存在正常数$c,n_0$和函数$T,g$。使得当$N \geq n_0$有$T(N) \geq cg(N)$，则记作$T(N) = \Omega (g(N))$

我们使用刚才的反例，令$T = 3x$和$g = x$，只需要找任意的一个小于3的常数$c$，就有$T(x) \geq g(x)$

定义3： 我们说$T(N) = \Theta(h(N))$，这是指$T(N)$和函数$h(N)$都满足定义1和定义2。即：$T(N) = O(h(N))$和$T(N) = \Omega(h(N))$

若我们都选择$N\to \infty$，则这个时候在数学上叫同阶无穷小。

定义4：小o表示 若$T(N) = O(h(N))$但$T(N)\neq \Theta(h(N))$，则我们说$T(N) = o(h(N))$

在数学上，令$N\to\infty$则称其为高阶无穷小。

这样的分析促使我们其寻找一个更简便的方法，回看定义3，此时我们发现和$\epsilon -\delta$语言是类似的，也就是说这依然停留在数列极限的范畴上，也就是说该定义在$x\to\infty$可以重写为：

$\begin{aligned} \lim\limits_{x\to \infty} \frac{T(x)}{h(x)} = c \end{aligned}$

其中$c$是任意的不等于0的常数。而当$c=0$时，则是定义4的结果。

我们费尽心思定义的这些是为了估算在编程中函数运算的“时间”数量级有多大，当我们给定任意两个函数，也许会存在一些点，在这些点上使得一个函数的值小于另一个函数。所以一般情况下的比较$f(x) < g(x)$是没有意义的，我们一般考虑函数的相对增长率。

如同我刚才所说，当$N$较小时，$1000N$比$N^2$是要大的。但考虑到在某些点后，函数就会爆发式的增长。在实际编程中，这些点就是我们代码执行的步数情况。考虑一些较小的数据通常是没有意义的。所以引入定义来做比较度量算法的能效性是有用的。

当我们说$N= O(N^2)$时，实际上是在说$N$的增长不比$N^2$快，所以此时我们说$N^2$是$N$的上界。但他不一定是比较好的上界，因为他们相差太大了。另外，若$f(N) = \Omega(T(N))$，同样的我们可以说$T(N)$是$f(N)$的一个下界，意味着每个$f(N)$在$N$之后都大于$T(N)$的值。

接下來拓展一下。

法则1

若$T_1(N) = O(f(N))$且$T_2(N) = O(g(N))$，那么

  • $T_1(N) +T_2(N) = \max \{O(f(N)) ,O(g(N))\}$
  • $T_1(N) \times T_2(N) =O(f(N) * g(N))$

法则2

若$T(N)$是$k$次多项式，则$T(N) = \Theta(N^k)$

法则3

对任意常数$k$，$\log^k N = O(N)$。

对法则1,我们设$O(f(N)) = x^3$，$O(g(N)) - x^4$，可以看出$x^3 $在$x=1$之后有$x^3 < x^4$1。所以是$x^3 = O(x^4)$。那么$T_1(N)+T_2(N) = O(x^4)+O(x^3) = O(x^4)$，也就是定义$\max\{O(f(N)),O(g(N))\}$。对第二个法则，$x^3 \times x^4 =x^7$，所以$T_1(N) T_2(N) = x^3x^4=x^7$

我们可以皆由推广到任意的多项式。设$O(f(x)) = kx^3+sx^2+vx+e$是一个3次多项式。令$O(g(x)) = lx^4+ax^3+bx^2+cx+d$三一个任意的4次多项式，其中$k,s,v,e,l,a,b,c,d$是任意的常数。利用定义1,在一个足够大的$N$后，我们选择比$k$大的常数$k’$和$l$的常数$l’$，就有$k’x^3 \geq O(f(x)) \Rightarrow k’x^3 = \Omega(f(x))$和$l’x^4 \geq O(g(x)) \Rightarrow l’x^4 = \Omega (g(x))$。这些都可以通过取$x\to\infty$求其极限得到。

所以自然的对法则2,k次多项式的一个好的估算就是$\Theta(x^k)$。

最后，我们证明对常数$k,\log^kN = O(N)$：

考虑极限

$\begin{aligned} \lim\limits_{x\to\infty}\frac{\log^k(x)}{x} = \frac{k\log(x)}{x}\overset{I'H}{==}\frac{k*1/x}{1} = k*1/x =0 \end{aligned}$

其中$I’H$是洛必达定理的缩写。

所以，当你不确定函数的相对增长率时，你就可以通过计算极限$\lim\limits_{x\to\infty}f(x)/g(x)$来确定f和g的相对增长率。极限是0,就是定义4的情况，极限是任意常数，则它是定义3的情况。若极限是$\infty$，则$g(x) = o(f(x))$

最大子序列和问题的解

我们用一些例子来叙述如何解决最大子序列问题，首先是一个暴力计算的方法：

该算法可以很快的估算出时间为$O(N^3)$。这取决12行厘米的$O(1)$语句，第三个循环的大小是$j-i+1$。所以总数不超过N，全部和起来就是$O(1\cdot N\cdot N\cdot N) = O(N^3)$。但实际上，更精确的上界是$\Theta(N^3)$。我们考虑最特殊的情况，一个全是1的序列。那么他的最大子序列就是$\sum^{N-1}_{i=0}\sum^{N-1}_{j=i}\sum^j_{k=i} 1 $，为了求解这个级数的和，首先注意到$\sum^{j}_{k=i}1$，它是$j-i+1$。

接着，对第二个求和有

$\begin{aligned} \sum^{N-1}_{j=i} (j-i+1) = \frac{(N-i+1)(N-i)}{2} \end{aligned}$

这里我们用到了等差数列的求和公式，注意每一项都是前一项+1。那么末项是$(N-1)-i+1 = N-i$，首项就是$i-i+1=1$。用等差数列的求和公式就有上述的式子了。

最后我们处理剩下的求和符号。

注意

$\begin{aligned} & \sum_{i=0}^{N-1}\frac{(N-i+1)(N-i)}{2}=\sum_{i=1}^{N}\frac{(N-i+1)(N-i+2)}{2} \\ & =\frac{1}{2}\sum_{i=1}^{N}i^{2}-\left(N+\frac{3}{2}\right)\sum_{i=1}^{N}i+\frac{1}{2}(N^{2}+3N+2)\sum_{i=1}^{N}1 \\ & =\frac{1}{2}\frac{N(N+1)(2N+1)}{6}-\left(N+\frac{3}{2}\right)\frac{N(N+1)}{2}+\frac{N^{2}+3N+2}{2}N \\ & =\frac{N^{3}+3N^{2}+2N}{6} \end{aligned}$

但我们可以做的更好，第一个序列需要计算所有子序列的和其比较，我们改进的算法如下，它是$O(N^2)$的。具体思路如下：我们放弃计算所有的子序列，注意在我们按照顺序去计算序列的时候，我们也许会作很多重复的事情，例如$A_2+A_3$首先会在$i=2$的时候$j=3$计算一次，在$j>3$的时候又会计算一次，这很大程度上浪费了计算机的资源，因此我们接下来就是为了避免重复计算。

只需要遍历一次从$j=i$为起点，以$N-1$为终点，先得到一个极大和，若序列的下一项加进去比原来的大，则替换极大和。直到遍历完整个序列。接着令部分和为0从新开始遍历然后和极大和作比较。不算是很聪明的代码，依然有很多的重复，但比上一个好多了。
